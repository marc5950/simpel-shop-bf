# Teknisk Dokumentation for simpel-shop-bf

## Komponenter

Komponenter er defineret inden projektets start. Alle komponenter sættes ind i "component"-mappen, så de allerede er klar til at laves, når kodningen går i gang.

### Navngivning af komponenter

Komponenter navngives som et sammenhængende ord, uden brug af bindestreg eller andre specialtegn. Forbogstav laves stort. Hvis navnet består af to ord, laves forbogstavet på begge ord til stort.

## Branches

Når man koder, opretter man en branch først, som man redigere i, dette er for at skabe så få problemer som muligt. Main bruges kun til at merge branches ind i. Branch navne består kun af små bogstaver og mellemrum defineres med en bindestreg(-). Branch navne skal være på engelsk.

## Commit beskeder

Commit beskeder skal ske på engelsk. Beskederne behøver ikke være lange, men skal beskrive hvilken handling der er lavet. Dette gøres ved at skrive added, removed, fixed...
"Added..." skrives foran ting der er tilføjes, "Removed..." foran ting der er fjernet og "Fixed... foran ting der er fixet.
